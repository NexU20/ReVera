Slide 3 — Rencana Pengumpulan Data

Dua sumber data yang saling melengkapi

// sumber A — objek utama

Ulasan riil 5 produk terlaris

Diambil langsung dari toko target lewat web scraping memakai Playwright, karena halaman Shopee dimuat dinamis (JavaScript) dan tidak punya API publik. Pakai jeda waktu acak agar mirip pola manusia dan tidak membebani server.

// sumber B — pemerkaya data latih

Dataset publik (Kaggle)

Dipilih dari kategori sejenis untuk menambah contoh kelas negatif & netral yang biasanya langka. Produk terlaris cenderung didominasi ulasan positif, jadi data publik ini menyeimbangkan kelas.

// etika & legal

Mengikuti ketentuan layanan (ToS): volume diambil seperlunya, hanya konten ulasan & rating, mengabaikan data pribadi pengguna, dan murni untuk keperluan akademik non-komersial.

Plan B
Kalau scraping diblokir anti-bot, dataset publik jadi cadangan utama agar proyek tetap bisa jalan. Target minimal jumlah ulasan ditetapkan sejak awal.

Slide 5 — Rencana Pra-pemrosesan

Satu pembersihan awal, lalu dua jalur

case folding tangani huruf berulang emoji → teks kamus slang pemetaan dwibahasa (InSet)

JALUR BERAT untuk SVM

  • Tokenisasi, hapus stopword, dan stemming Sastrawi
  • Tujuannya memadatkan jumlah fitur TF-IDF
  • Membuat model lebih fokus ke kata bermakna

JALUR RINGAN untuk Regex

  • Tanpa stemming & tanpa stopword agresif
  • Menjaga kata kunci aspek tetap utuh
  • Agar pola Regex bisa mendeteksi keluhan secara akurat
// pelabelan hibrida

Sentimen dilabeli otomatis dari rating, lalu diverifikasi manual khusus bintang 3 yang paling ambigu (campuran pujian & keluhan). Ada aturan baku untuk kasus tepi: ulasan satu kata atau yang hanya berisi emoji.

// atasi data tidak seimbang

Karena positif mendominasi, contoh kelas minoritas (negatif & netral) ditambah dari dataset publik agar model tidak “buta” terhadap keluhan.

Slide 6 — Rencana Pemodelan

SVM untuk sentimen, lalu gerbang penyaring

// fitur

TF-IDF (Unigram + Bigram)

Bigram dipakai agar frasa seperti “tidak bagus” tertangkap utuh — bukan dipecah jadi kata terpisah.

// klasifikasi

Support Vector Machine

Dipilih karena andal pada fitur teks berdimensi tinggi dan lebih ringan dibanding deep learning. Output: positif / negatif / netral.

// gerbang

Gatekeeper

SVM jalan dulu, lalu sistem hanya meneruskan ulasan negatif dan bintang 3 ke tahap aspek. Ulasan positif dibuang demi efisiensi.

// kenapa gatekeeper penting

Strategi ini menangkap anomali: ulasan bintang tinggi tapi teksnya berisi keluhan nyata. Jadi keluhan tidak lolos hanya karena ratingnya bagus.

// pembagian data

Stratified split + cross-validation. Sebagian data toko yang sudah dilabeli disuntikkan ke data latih publik, agar model menyesuaikan diri dengan gaya bahasa toko target.

Catatan
Penandaan aspek (kualitas barang, kemasan, pengiriman) memakai Regex berbasis aturan pada teks jalur ringan — transparan dan mudah ditelusuri kenapa sebuah ulasan masuk aspek tertentu.
Slide 7 — Rencana Evaluasi

Mengukur dengan jujur, bukan cuma akurasi

SENTIMEN (SVM) — kuantitatif

  • Precision, Recall, F1 dilaporkan per kelas
  • Macro-Average F1 agar kelas minoritas (negatif & netral) berbobot setara
  • Sengaja hindari accuracy — bisa menyesatkan karena positif mendominasi
  • Confusion matrix untuk melihat pola salah klasifikasi, terutama kelas netral & bintang 3

ASPEK (Regex) — kualitatif

  • Diperiksa manual pada sampel ulasan
  • Menguji apakah pola Regex menandai aspek dengan tepat
  • Mengecek apakah ada keluhan yang terlewat atau salah aspek
// kriteria sukses

(1) Macro-F1 cukup baik dalam mengenali kelas minoritas, dan (2) pemetaan aspek keluhan jelas serta bisa ditindaklanjuti.

Realistis
Kami sadar kelas netral akan jadi sumber error terbesar dan Regex punya keterbatasan pada variasi kata — ini akan kami akui terbuka, bukan disembunyikan.

Slide 9 — Risiko & Mitigasi

Antisipasi yang sudah disiapkan

Scraping diblokir anti-bot / kendala ToS
Jeda waktu acak, batasi volume seperlunya, dan siapkan dataset publik sebagai cadangan.
Kelas tidak seimbang & bintang 3 ambigu
Suntik data minoritas dari dataset publik + pelabelan hibrida dengan aturan baku untuk kasus tepi.
Domain mismatch & performa netral turun
Vocabulary fusion pada TF-IDF + suntik data lokal terlabel; kelemahan dipetakan lewat confusion matrix.
Regex gagal menangkap variasi kata
Pola dirancang mencocokkan teks pasca-normalisasi slang, dan keandalannya diuji manual pada sampel.
